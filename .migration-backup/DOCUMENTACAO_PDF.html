VISÃO GERAL EXECUTIVA

O Que é Quanta Flow?

Quanta Flow é uma plataforma SaaS de automação de vendas que:

  • ✅ Centraliza comunicação de múltiplos canais (WhatsApp, Telegram, Instagram, E-mail)
  • ✅ Detecta intenção de compra via IA (OpenAI GPT-4)
  • ✅ Automatiza respostas e fluxos de vendas
  • ✅ Gerencia fila de atendimento com SLA em tempo real
  • ✅ Integra com sistemas externos (Google Sheets, Zapier, webhooks)
  • ✅ Oferece análise completa de performance
  • ✅ Suporta múltiplos usuários com RBAC granular

DATABASE SCHEMA (20+ TABELAS)

Tabelas Principais

1. users - Autenticação

  • id: UUID (PK)
  • email, password: bcrypt hashed
  • status: active, inactive, suspended
  • tokenVersion, mustChangePassword: JWT management

2. unified_contacts - CRM Central

  • pipelineStage: novo, contatado, qualificado, convertido
  • temperature: frio, morno, quente
  • score, lastIntent, tags, notes
  • queueStatus: waiting, assigned, resolved [fila]
  • slaDeadline, activeFlowId: automação integrada
  • assignedToUserId: agent assignment

3. conversations - Chats Multi-Canal

  • channel: 'whatsapp' (default), 'telegram', 'instagram', 'email' [Sprint 5]
  • remoteJid: ID do contato no canal
  • lastMessage, lastMessageAt, unreadCount

4. messages - Histórico

  • direction: incoming, outgoing
  • content, mediaType, mediaUrl
  • timestamp

5. automation_flows - 9 Seções

  • triggerKeywords: gatilhos para acionamento
  • responseTemplate, responseDelay: resposta automática
  • systemPrompt, temperature: IA config
  • successCondition, interruptCondition: lógica condicional
  • steps (JSONB): multi-step sequences
  • conditionalExits (JSONB): transferência entre fluxos

6. outbound_webhooks [Sprint 5]

  • events: lead.created, lead.qualified, flow.success, flow.interrupt, conversation.closed
  • secret: HMAC-SHA256 key
  • lastStatus, lastTriggeredAt: tracking

7. sheet_integrations [Sprint 5]

  • spreadsheetId, sheetName: Google Sheets reference
  • triggerEvent: lead.created, lead.qualified, etc
  • columnMapping (JSONB): campo → coluna
  • googleToken: OAuth2 token (encrypted)

8. email_configs [Sprint 5]

  • smtpHost, smtpPort, smtpUser, smtpPass: credenciais
  • imapHost, imapPort: receção (opcional)
  • isActive: status

9-13. Tabelas Adicionais

  • learning_tracks: microlearning CRUD
  • roles, permissions, user_roles, role_permissions: RBAC (3 roles, 18 permissions)
  • audit_logs: rastreamento de ações
  • agent_assignments: atribuição a agentes
  • omnichannelMessages: intent tracking por canal
  • quick_replies, branding_config, api_configs, evolution_configs
  • learning_deliveries, pipeline_stages, channels, settings

FLUXOS DE DADOS

Fluxo 1: Mensagem WhatsApp → Automação

1. Z-API/Baileys recebe msg
   ↓
2. POST /api/whatsapp/webhook
   ↓
3. processIncomingMessage() salva em BD
   Socket.io emite para UI em tempo real
   ↓
4. OpenAI detecta intenção + score
   ↓
5. messageProcessor() busca automationFlow
   Por keyword trigger ou activeFlowId
   ↓
6a. Se interruptCondition → Entra na fila
    queueStatus = 'waiting'
    SLA deadline = now + slaMinutes
    ↓
6b. Se successCondition → Resolve
    dispatchEvent('flow.success')
    ↓
6c. Se steps → Agenda mensagens
    JobQueue.add() a cada 5s
    ↓
7. Envio via sendChannelMessage()
   Z-API / Telegram / Email / Instagram

Fluxo 2: SLA Queue Tracking

Lead entra na fila → Timer SLA inicia
   ↓
JobQueue verifica deadline a cada 5s
   ↓
Frontend mostra countdown em tempo real
   ↓
Agent clica Assumir/Resolver
   ↓
Status atualizado via Socket.io

Fluxo 3: Lead → Google Sheets

Lead stage = 'qualificado'
   ↓
dispatchEvent('lead.qualified')
   ↓
webhookDispatcher() processa
   ↓
Google Sheets API append row
   ↓
Planilha atualizada em tempo real
