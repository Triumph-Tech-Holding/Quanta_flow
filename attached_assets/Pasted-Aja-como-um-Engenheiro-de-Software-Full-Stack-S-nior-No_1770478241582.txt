Aja como um Engenheiro de Software Full Stack Sênior. Nossa missão é transformar o sistema atual em um Conversational CRM Omnichannel de alta performance, eliminando gargalos de cache e unificando 6+ canais de atendimento em uma única "Single Source of Truth".

🛠️ STACK TÉCNICO
Backend: Node.js + Express.js.

Frontend: React + Vite.

Banco de Dados: PostgreSQL + Drizzle ORM.

Comunicação: Socket.io (Real-time) + Webhooks.

IA: OpenAI API (GPT-4) para detecção de intenções.

WhatsApp: Manter suporte híbrido Z-API (atual) e Evolution API.

1️⃣ ESTABILIDADE E ANTI-CACHE (SOLUÇÃO IMEDIATA)
Implemente um middleware global de Anti-Cache para resolver o erro 304 Not Modified que está travando a atualização do dashboard.

Configure os headers: Cache-Control: no-store, no-cache, must-revalidate, proxy-revalidate, Pragma: no-cache e Expires: 0.

Aplique este middleware em todas as rotas de /api/v1/inbox, /settings e /webhooks.

2️⃣ SCHEMA DE DADOS OMNICHANNEL (DRIZZLE ORM)
Crie/Atualize as tabelas para suportar a unificação de leads:

channels: Suporte para whatsapp, instagram, facebook, linkedin, youtube, tiktok e x.

unified_contacts: Chave mestre do lead (nome, avatar, email e telefone unificados).

contact_identifiers: Vincula IDs de redes sociais (ex: @user do Insta ou ID do TikTok) ao contato unificado.

omnichannel_messages: Registro de mensagens com campos para channel_id, detected_intent (IA) e message_type.

lead_pipeline_stages: Estrutura do funil estilo Kommo (Novo, Qualificado, Proposta, Fechado).

lead_temperature: Classificação automática (Frio, Morno, Quente) baseada na IA.

3️⃣ INTELIGÊNCIA ARTIFICIAL E FLOW TRIGGERS (ESTILO MANYCHAT)
Crie o intentDetectionService.js:

Lógica: Ao receber qualquer mensagem, envie o conteúdo + as últimas 3 mensagens (contexto) para a OpenAI.

Classificação: Detectar intenções como compra_quente, dúvida, reclamação.

Ação: Se compra_quente, atualize automaticamente o estágio do lead no CRM para "Qualificado" e a temperatura para "Quente".

4️⃣ ADAPTADORES OMNICHANNEL (YOUTUBE, TIKTOK, X, META)
Crie um roteador central de Webhooks em /api/v1/webhooks/omnichannel:

WhatsApp: Manter integração Z-API ativa.

Instagram/FB: Via Meta Graph API.

YouTube/TikTok/X: Implementar adaptadores para normalizar comentários e DMs para o formato padrão do Inbox.

Normalização: Se um lead enviar um Zap (telefone) e depois um Direct (username), use o serviço de unificação para mesclá-los se houver um dado em comum (email ou telefone).

5️⃣ FRONTEND: INBOX & VISUAL PIPELINE
Atualize a interface do usuário:

Visual Pipeline (CRM): Crie um componente de funil onde o atendente pode arrastar o lead entre colunas de venda.

Inbox Sidebar: Exiba o "Cartão do Lead" com todos os canais conectados dele e a temperatura atual.

Real-time: Use Socket.io para que mensagens de qualquer rede social apareçam no Inbox em < 2 segundos sem Refresh.

Identidade Visual: Garanta campos para upload de Logo e Favicon na aba "Geral" das configurações.

❗ REQUISITOS CRÍTICOS DE ENTREGA
Deduplicação: Implementar lógica para evitar contatos duplicados no banco.

Segurança: Criptografar tokens de acesso das redes sociais.

Logs: Adicionar logs detalhados para cada Webhook recebido para facilitar o debug.

Feedback Visual: O indicador de "Conectado" no WhatsApp deve refletir o status real da Z-API.
