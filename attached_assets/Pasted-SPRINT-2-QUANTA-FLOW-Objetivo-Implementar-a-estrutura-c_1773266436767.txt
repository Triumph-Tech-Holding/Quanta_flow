SPRINT 2 — QUANTA FLOW
Objetivo: Implementar a estrutura completa das 7 partes do assistente de IA —
atualizar o schema do banco e criar a tela de configuração no frontend.

=== CONTEXTO DO SISTEMA ===

Stack:
- Backend: Node.js + Express + Drizzle ORM
- Banco: PostgreSQL (produção) — 49 contatos reais, NÃO pode quebrar
- Frontend: React + Vite + TypeScript + Shadcn UI + Tailwind CSS
- API: REST via React Query v5 + WebSocket via Socket.io
- Deploy: Replit (produção separado do desenvolvimento)
- ORM: Drizzle — todas as mudanças de schema via migration

=== O QUE JÁ EXISTE E FUNCIONA (NÃO ALTERAR) ===

1. Tabela automation_flows com campos:
   - name, triggerKeywords, responseTemplate, isActive
   - CRUD completo: getAutomationFlowsByUser, createAutomationFlow,
     updateAutomationFlow, deleteAutomationFlow
   - findMatchingAutomationFlow(userId, message) no storage.ts

2. Frontend: Página /automation com interface CRUD funcional
   - Usa Shadcn UI + React Query
   - Já tem formulário de criação/edição de fluxos

3. Banco PostgreSQL rodando em produção com Drizzle ORM

=== TAREFAS EXATAS A EXECUTAR ===

--- TAREFA 2.1 — ATUALIZAR SCHEMA DO BANCO ---

Adicionar os seguintes campos à tabela automation_flows via migration Drizzle:

  systemPrompt        text          (nullable, default: null)
  initialMessage      text          (nullable, default: null)  
  temperature         numeric(3,2)  (nullable, default: 0.4, range: 0.0–1.0)
  responseDelay       integer       (nullable, default: 10)
  inactivityTimeout   integer       (nullable, default: 10)
  successCondition    text          (nullable, default: null)
  interruptCondition  text          (nullable, default: null)
  summaryEnabled      boolean       (nullable, default: false)
  summaryFields       text          (nullable, default: null)

  a) Todos os campos devem ser NULLABLE para não quebrar os registros
     existentes em produção
  b) Gerar e aplicar a migration via Drizzle (drizzle-kit generate + push)
  c) Confirmar que os 49 contatos e fluxos existentes não foram afetados
  d) Atualizar o tipo TypeScript do modelo automation_flows no schema.ts

--- TAREFA 2.2 — ATUALIZAR O BACKEND ---

Na página /automation, expandir o formulário de criação/edição de fluxos
para incluir as 7 partes do assistente, organizadas em seções:

SEÇÃO 4 — Modelo e Criatividade
