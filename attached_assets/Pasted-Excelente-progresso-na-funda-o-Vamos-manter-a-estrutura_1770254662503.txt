Excelente progresso na fundação. Vamos manter a estrutura de pastas atual e o uso do Drizzle ORM. Agora, inicie a construção do Módulo 1: Inbox Omnichannel.

OBJETIVO:
Permitir que o usuário conecte seu WhatsApp via Evolution API v2 e visualize as mensagens recebidas em tempo real.

STACK MANTIDO:
- Backend: Node.js + Express.js
- Frontend: React + Vite
- Banco de Dados: PostgreSQL
- ORM: Drizzle (mantém o padrão da Fase 1)
- WebSocket: Socket.io (mesma porta da API)
- API Externa: Evolution API v2

TAREFAS DE BACKEND:

1. Schema (Drizzle):
   - Adicione as tabelas ao arquivo de schema existente (não crie novo arquivo)
   - Tabelas: evolution_configs, conversations, messages
   - Use UUIDs e timestamps (padrão da Fase 1)
   - Gere e execute a migração

2. Evolution Service:
   - Crie arquivo: backend/src/services/evolutionService.js
   - Funções:
     • createInstance(userId, evolutionUrl, globalToken)
     • getQRCode(instanceId)
     • sendMessage(instanceId, phoneNumber, message)
     • validateToken(token)
   - Use axios para chamadas HTTP
   - Adicione tratamento de erros com try/catch

3. Webhook para Receber Mensagens:
   - Crie endpoint: POST /webhooks/evolution
   - Validar header "Authorization" com global_token
   - Processar evento "messages.upsert"
   - Salvar mensagem no banco (tabela messages)
   - Atualizar conversation (last_message, last_message_at)
   - Emitir evento Socket.io: "message:received"

4. Socket.io Configuration:
   - Configure na mesma porta da API (ex: 3000)
   - Eventos a emitir:
     • "message:received" - nova mensagem
     • "message:sent" - mensagem enviada
     • "instance:connected" - WhatsApp conectado
   - Validar user_id em cada evento (segurança)

5. URL Dinâmica do Replit (CRÍTICO):
   - Use a variável nativa: process.env.REPLIT_DEV_DOMAIN
   - Construa a BASE_URL assim:
     const BASE_URL = `https://${process.env.REPLIT_DEV_DOMAIN}`;
   - Webhook URL será: `${BASE_URL}/webhooks/evolution`
   - Registre automaticamente na Evolution API ao conectar
   - Se REPLIT_DEV_DOMAIN não existir, use fallback: process.env.BASE_URL (do .env)
   - Adicione log: console.log(`Webhook URL: ${BASE_URL}/webhooks/evolution`)

TAREFAS DE FRONTEND:

1. Tela de Configuração (Inbox Settings):
   - Componente: frontend/src/components/Inbox/EvolutionConfig.jsx
   - Campos:
     • Input: "URL da Evolution API"
     • Input: "Global Token"
     • Botão: "Conectar WhatsApp"
   - Ao clicar, chama POST /api/evolution/connect
   - Exibe modal com QR Code em Base64

2. Modal de QR Code:
   - Componente: frontend/src/components/Inbox/QRCodeModal.jsx
   - Exibe QR Code em Base64
   - Polling a cada 5 segundos: GET /api/evolution/status
   - Quando status = "connected", fecha modal automaticamente
   - Timeout após 60 segundos (QR Code expira)

3. Visualização do Chat:
   - Componente: frontend/src/components/Inbox/ChatWindow.jsx
   - Layout:
     • Esquerda: Lista de conversas (ConversationList.jsx)
     • Direita: Janela de chat (ChatWindow.jsx)
   - Listar conversas com último contato
   - Clicar em conversa abre histórico
   - Campo de input para enviar mensagens
   - Usar socket.io-client para receber mensagens em tempo real

4. Integração Socket.io:
   - Conectar ao servidor na inicialização
   - Escutar eventos:
     • "message:received" - atualizar chat
     • "instance:connected" - atualizar status
   - Emitir evento ao enviar mensagem: "message:send"

3. Socket.io na Mesma Porta:

4. Banco de Dados:
